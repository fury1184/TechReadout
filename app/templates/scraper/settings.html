Open WebUI Automatic Lookup
{# Re-submit the other cards' fields so they aren't wiped on this save #}
When enabled, Open WebUI runs after Scrape.Do and before manual AI Import in the lookup chain. Open WebUI results are never auto-accepted — they always go to the Pending Review queue for a quick human check, no matter how confident the match looks.

Credential status:

  • OPENWEBUI_API_TOKEN: {% if openwebui_token_present %} present {% else %} missing {% endif %}
Set this in docker-compose.yml under the app service environment. Generate it in Open WebUI under Settings → Account → API Keys.
What changed in v{{ APP_VERSION }}

The free scraper chain (FlareSolverr, Playwright TechPowerUp, Playwright Amazon, Intel ARK, AMD Official, manufacturer sites) was retired in v3.0 because Cloudflare and Amazon anti-bot measures had made every free path unreliable.

Lookup chain is now: seed DB → Scrape.Do (paid) → Open WebUI (optional, self-hosted LLM) → manual AI Import. The seed database covers most common hardware out of the box, so paid and LLM lookups are rare in practice.

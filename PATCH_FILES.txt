TechReadOut 3.5.8 - Installed Status Synchronization Hotfix

Apply over TechReadOut 3.5.7.

Changed files:
- app/version.py
- app/inventory_rules.py
- app/routes/main.py
- app/routes/backup.py

Fixes:
- Assigning an inventory item to a host from the Edit Inventory page now
  automatically sets its status to Installed.
- The dedicated Assign to Host workflow uses the same centralized rule.
- Imported inventory assigned to a host is normalized to Installed.
- Unassigned inventory statuses are not changed automatically.

No database migration is required.

Apply:
1. Extract this archive.
2. Copy/overlay its files into /opt/stacks/techreadout.
3. Rebuild and recreate the application container:

   cd /opt/stacks/techreadout
   docker compose up -d --build --force-recreate

Verify:
   docker compose exec app python -c "from app.version import APP_DISPLAY_VERSION; print(APP_DISPLAY_VERSION)"

Expected:
   TechReadOut v3.5.8

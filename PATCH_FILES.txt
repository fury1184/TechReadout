TechReadOut 3.5.7 — Review Match Selection Hotfix

Apply over TechReadOut 3.5.6.

Changed files:
  app/templates/inventory/add.html
  app/routes/api.py
  app/scrapers/lookup.py
  app/version.py

Review modal fixes (app/templates/inventory/add.html):
- Candidate cards now update the JavaScript selection state reliably.
- Avoids recursive/synthetic label/radio click behavior.
- Automatically selects the highest-ranked candidate.
- Confirm button falls back to the checked radio selection.
- Shows a clear warning if a candidate has no saved spec ID.

Lookup accuracy fixes:
- app/routes/api.py: auto-accept now requires validate_result() to confirm the
  match, so Strategy 4 fuzzy recall can no longer auto-accept a similar-but-wrong
  part (e.g. a different Xeon SKU) on shared generic tokens.
- app/scrapers/lookup.py: Scrape.Do TechPowerUp requests now pass render=true so
  JS-rendered spec pages resolve reliably.

After copying the files into /opt/stacks/techreadout:
  docker compose up -d --build --force-recreate

Then hard refresh the browser with Ctrl+F5.
